                              malcolmboura • 2 years ago • edited

                              Very helpful but I have some observations.

                              Posix states that a struct termios passed to tcsetattr() must have been initialised by a call to tcgetattr() or behaviour is undefined. Hence code pattern is:
                              tcgetattrr(...);
                              // modify as necessary
                              tcsetattr(...)
                              Gnu says much the same but with more explanation. http://www.gnu.org/software...

                              I didn't find any statement regarding setting a flags member to zero (eg tty.c_cflag = 0) so, for both the reasons given by gnu and others, it is unsafe to assume that it can be.

                              The character size flags are, at least in the termios.h on my linux machine, bitfields. Hence setting this value must be done by first clearing the bits and then setting them. eg
                              tty.c_cflag &= ~CSIZE;
                              tty.c_cflag |= CS6;
                              CSIZE is a puzzle. It appears to be a mask and not a size.

                              I am surprised that the header file does not say anything much about the baud rate bit patterns. The gnu advice is to use the functions to set/get the speed, not the flags.
